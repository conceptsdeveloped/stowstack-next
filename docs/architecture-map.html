01Authentication Routing

Four independent systems. Clerk wraps everything as passthrough; real gating happens in the other three + API key + cron secret.

flowchart TB
  req[Incoming Request]
  req --> mw[Clerk middleware
marks routes public]
  mw --> router{Route prefix}

  router -->|/admin/* or /api/admin-*| ak[requireAdminKey
X-Admin-Key header]
  router -->|/portal/* or /api/client-*| pac[Access Code
Bearer token]
  router -->|/partner/* or partner API| os[getSession
ss_ token · 30d TTL]
  router -->|/api/v1/*| vk[v1-auth
partner API key]
  router -->|/api/cron/*| cs[verifyCronSecret
CRON_SECRET]
  router -->|public| pass[no auth]

  ak --> okAdmin((Admin route))
  pac --> okPortal((Portal route))
  os --> okPartner((Partner route))
  vk --> okV1((V1 route))
  cs --> okCron((Cron route))
  pass --> okPub((Public route))

  ak -. fail .-> rej[401]
  pac -. fail .-> rej
  os -. fail .-> rej
  vk -. fail .-> rej
  cs -. fail .-> rej

  TRIG{Trigger}
  TRIG -->|lead capture| E1[partial_leads]

  TRIG -->|moved out| E3[moveout webhook]
  TRIG -->|audit generated| E4[post-audit email]

  E1 --> F[funnelConfigToDripSteps]
  E2 --> RECOV[recovery template]
  E3 --> MOV[moveout_remarketing]
  E4 --> NUR[nurture_enrollments]

  F --> DS[drip_sequences
status=active
next_send_at=now+delay0]
  RECOV --> DS
  MOV --> DS
  NUR --> NS[nurture_sequences]

  CRON1[/process-drips 5 AM/] --> DS
  CRON2[/process-nurture 6 AM/] --> NS
  CRON3[/process-recovery 7 AM/] --> DS

  RES --> UPD[Advance step

  TWI --> UPD
